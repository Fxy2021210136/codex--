# 功能说明与登录差异

本文说明当前版本已经具备的功能，以及“访客模式 / 登录用户 / 管理员”三种使用状态的区别。

## 当前核心功能

### 1. 通用工程计划

- 支持房建工程、道路工程、桥梁工程、市政管网、工业安装和自定义工程。
- 支持总控计划和工序计划两种粒度。
- 支持自定义空间维度，例如楼栋/楼层、标段/区段、桥梁/墩台、片区/管段、装置区/系统。
- 可从项目描述生成项目条件、工序计划、关键节点和风险摘要。

### 2. 专业排程逻辑

- 支持 FS / SS / FF / SF 四类逻辑关系。
- 支持 Lag 延迟。
- 支持 CPM 正排、逆排、总时差和关键路径。
- 支持循环依赖校验、锁定节点可达性校验。
- 支持施工日历、周末规则、停工期和节假日策略。
- 支持资源负荷计算、超载识别和自动错峰。

### 3. 工序模板库

- 内置多工程类型模板包。
- 支持新增、复制、编辑、删除自定义工序模板。
- 模板可配置项目类型、阶段、专业、工期、前置关系、Lag、资源、材料节点和空间展开维度。
- AI 候选模板可由用户确认后写入自定义模板库。

### 4. 项目工作台

- 横道图、AON 网络图、WBS、工序详情、工序库、资源负荷、计划审查、关键节点和汇报版。
- 支持任务日期、状态、进度、实际日期、工作面、工程量、资源投入和现场状态编辑。
- 支持多版本计划基准、偏差对比、后续任务顺延、撤销和重做。
- 支持 Excel 五表工作簿导出、CSV 导出、项目 JSON 备份/恢复和 A4 横向打印。

### 5. 后端与数据库

- Python 后端提供项目、账号、模板、AI 配置和运营接口。
- 默认 SQLite 数据库为 `data/app.db`。
- 支持本机启动、备份、恢复、Tunnel 临时公网访问和 Docker 部署准备。

### 6. AI 与 Codex

- 支持 DeepSeek、Gemini 和 OpenAI Responses API 的服务端代理。
- API Key 只保存在服务端，不进入浏览器。
- 支持模型连接诊断、AI 调用限流和每日额度。
- Codex 智能体为管理员专用，默认关闭，默认只读沙箱。

## 登录与不登录的区别

| 能力 | 访客模式 | 登录用户 | 管理员 |
|---|---|---|---|
| 新建和编辑计划 | 支持 | 支持 | 支持 |
| 横道图 / 网络图 / WBS / 资源 / 审查 | 支持 | 支持 | 支持 |
| Excel / CSV / JSON 导出 | 支持 | 支持 | 支持 |
| 项目保存 | 可用，但依赖匿名浏览器 ID 或本机 local | 绑定账号保存 | 可查看自己的项目，并可不限额 |
| 跨浏览器 / 跨设备继续使用 | 不稳定 | 登录同一账号后可继续 | 登录管理员账号后可继续 |
| 自定义模板库 | 可试用，归属访客 ID | 绑定账号长期沉淀 | 绑定管理员账号长期沉淀 |
| 项目和模板隔离 | 浏览器级或本机级隔离 | 用户级隔离 | 用户级隔离并具备管理能力 |
| 修改密码 | 不适用 | 支持 | 支持 |
| 导出账号数据 | 不适用 | 支持 | 支持 |
| 删除账号和服务端数据 | 不适用 | 支持 | 支持 |
| 项目数量配额 | 可能受访客配额限制 | 受普通用户配额限制 | 默认豁免 |
| AI 每日额度 | 受访客额度限制 | 受用户额度限制 | 默认豁免 |
| AI Key 配置 | 不允许 | 不允许 | 支持 |
| 运营概览 | 不允许 | 不允许 | 支持 |
| 上线健康检查 | 不允许 | 不允许 | 支持 |
| Codex 智能体调用 | 不允许 | 不允许 | 管理员专用 |

## 推荐使用方式

### 临时体验

可以不登录，直接生成计划、编辑工序、导出 Excel。适合演示和快速试用。

### 正式保存项目

建议注册并登录。登录后项目、自定义模板和账号数据会绑定到当前账号，后续更稳定。

### 系统配置和运维

使用 `ADMIN_EMAILS` 中配置的邮箱登录，或使用 `ADMIN_TOKEN`。管理员负责 AI 配置、上线健康检查、运营概览和 Codex 能力边界。

## 当前边界

- 当前账号系统是轻量实现，尚未包含邮箱验证和找回密码。
- 长期多人使用建议部署在有持久磁盘的平台，或后续迁移 PostgreSQL。
- GitHub Pages 只能托管静态前端，不能单独承载登录、数据库、AI 代理和 Codex 后端。
- 公共站点不建议开放 Codex 写入权限。
